## ADDED Requirements

### 需求:MFCP 必须支持独立训练与验证入口
系统必须提供 MFCP 专用的数据集选择配置与训练入口，执行 MFCP 训练时必须只加载 `dataset/mfcp` 数据，禁止与 USTC/CIC 交替喂数或混合训练。

#### 场景:启动 MFCP 训练
- **当** 用户按配置指定数据集为 `mfcp` 并启动训练
- **那么** 训练与验证数据加载器必须仅读取 `dataset/mfcp` 路径，且运行日志中标记当前数据集为 `mfcp`

### 需求:MFCP 训练验证必须生成标准评估产物
系统必须在 MFCP 训练/验证流程中生成与既有数据集一致的输出产物，包括训练日志、指标曲线、混淆矩阵和评估报告。

#### 场景:完成一次 MFCP 训练验证
- **当** MFCP 训练和验证流程正常结束
- **那么** `outputs/logs/*.log`、`outputs/confusion_matrix_*.png`、`outputs/metrics_curve_*.png`、`outputs/report_*.md` 中必须出现对应 MFCP 运行结果

### 需求:新增支持不得改变 USTC/CIC 既有行为
系统在新增 MFCP 能力后必须保持 USTC 与 CIC 原有训练命令、配置默认值和产物结构兼容，禁止引入行为回归。

#### 场景:回归执行 USTC/CIC 训练
- **当** 用户使用历史命令运行 USTC 或 CIC 训练
- **那么** 训练流程、数据加载路径和输出文件结构必须与变更前一致

### 需求:README 必须提供 MFCP 可执行命令
系统文档必须新增 MFCP 的预处理、训练与验证命令示例，命令参数必须与代码入口一致并可直接执行。

#### 场景:按文档执行 MFCP 命令
- **当** 用户按照 README 中 MFCP 命令运行流程
- **那么** 预处理、训练与验证命令均可被正确解析并启动对应任务
